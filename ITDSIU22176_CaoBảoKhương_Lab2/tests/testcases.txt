DSA Lab 2 - Test Cases Description

1. MergeSort and QuickSort:
- Test 1 (Random + Duplicate): [5, 2, 9, 1, 5, 6]
- Test 2 (Already Sorted): [1, 2, 3, 4, 5]
- Test 3 (Reverse Sorted): [9, 8, 7, 6, 5]

2. Exercise 1 (Ticket Queue Sorting):
- Input: [(2,10:02,A), (1,10:05,B), (2,10:01,C)]
- Description: Tests stable sorting capability, maintaining creation time order when priorities tie.

3. Exercise 2 (Compare Advanced Sorts):
- Datasets are dynamically generated (N=1000): Random, Sorted, Reverse, and Many Duplicates to compare statistics.

4. Exercise 3 (Merge Two Sorted Lists):
- List 1: 1 -> 3 -> 7 -> null
- List 2: 2 -> 3 -> 9 -> null
- Expected: 1 -> 2 -> 3 -> 3 -> 7 -> 9 -> null

5. Exercise 4 (De-duplicate Contact List):
- Input: 5 -> 1 -> 5 -> 2 -> 1 -> null
- Expected: 5 -> 1 -> 2 -> null

6. Exercise 5 (Split Playlist):
- Input: 1 -> 2 -> 3 -> 4 -> 5 -> null
- Expected: 1 -> 2 -> 3 -> null and 4 -> 5 -> null

7. Exercise 6 (Stable Partition):
- Input: 3 -> 5 -> 8 -> 5 -> 10 -> 2 -> 1, x=5
- Expected: 3 -> 2 -> 1 -> 5 -> 8 -> 5 -> 10 -> null

8. Exercise 7 (Sort Linked List):
- Input: 4 -> 2 -> 1 -> 3 -> 4 -> 0 -> null
- Expected: 0 -> 1 -> 2 -> 3 -> 4 -> 4 -> null

9. Exercise 8 (Detect & Fix Cycle):
- Input: 1 -> 2 -> 3 -> 4 -> 5 -> 6 -> (back to 3)
- Expected: Finds cycle entry at 3, breaks it, returning linear list up to 6.

10. Exercise 9 (LRU Cache Simulation):
- Capacity: 3
- Operations: PUT(1), PUT(2), PUT(3), GET(1), PUT(4)
- Expected Cache State: 4, 1, 3 (MRU to LRU)
